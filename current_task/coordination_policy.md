# Task: Coordination Policy — Discord vs GitHub Issues

**Status:** Todo  
**Priority:** Medium  
**Goal:** Define clearly when agents and humans use Discord vs GitHub Issues for coordination, and document it in the workprocess docs.

---

## Proposed policy (to be confirmed)

| Situation | Channel |
|---|---|
| Quick questions, status updates, casual back-and-forth | Discord |
| Task assignment and tracking | GitHub Projects |
| Structured feedback on code or docs | GitHub PR comments |
| Bug reports, feature requests, decisions that need a record | GitHub Issues |
| PR review sign-offs | GitHub PR comments (role badges) |
